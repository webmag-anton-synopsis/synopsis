lazyloading
          
- подключаем lazyload.js
- инициализируем по определенному классу изображений (например .lazyload)
- картинкам указываем data-original с путями к оригиналам и в src - заглушку
  (лучше кодированую в base64 - https://webdesign-master.ru/services/lazy/).
  Картинкам нужно указать width/height, что бы не прыгали при отрисовке.
responsive image
          
<picture>
  <source media='(min-width: 1024px)' srcset='images/600x600.webp' type='image/webp'>
  <source media='(max-width: 1023px)' srcset='images/image-1x.webp images/image-2x.webp 2x'
          type='image/webp'>
  <img src='images/image-1x.png' alt='Just image'>
</picture>
          
        
  source в зависимости от ширины экрана (аттрибут media) загружает ту или
иную картинку.
  Это удобно, что бы для моб версии не грузить тяжелую картинку или, например,
если нужна другая картинка для моб. Или, например, для fallback'a для webp (хотя
возможно и обычный <img> c srcset для этого сгодится, где src будет fallback'ом
если не поддерживается webp формат из srcset).
  Если браузер не поддерживает MIME-тип из аттрибута type (например webp) или
нет совпадения с media-запросом, то вместо <source> берется следующий <source>
и так далее. Если ни один <source> не подошел - берется <img>

forms

* <input>, <button> могут быть валидно использованы без <form>.


  Значение атрибута action у <form> — это URL-адрес программы обработки данных, в которую будут
отправлены данные.

  Значением атрибута method у <form> могут быть только методы GET и POST.

  Каждый <input> также должно иметь атрибут name. Он предоставляет ключ, с которым можно
связать данные в базе данных. Например, если пользователь вводит данные 'William Preston' в input
с name='full_name', то данные будут сохранены в базе данных как {"full_name" : "William Preston"}.

  В отличие от элемента <input>, элемент <textarea> имеет закрывающий тег. Когда вы хотите
предварительно заполнить <textarea> данными, вы помещаете его между открывающим и закрывающим
тегами, вместо использования атрибута value, как вы делаете для <input type='text'>.

  <input type='checkbox' name='accept_terms' value='yes' checked>
По умолчанию данные, отправляемые на сервер из нажатого checkbox'a, имеют значение on.
Если вы присваиваете checkbox'у атрибут value='yes', то 'yes' будет отправлено на сервер
вместо on: {"accept_terms" : "yes"}.
Если атрибут value был опущен, то значение по умолчанию для ненажатого checkbox'а будет no,
т.е. отправленные данные будут {"accept_terms" : "no"}.

  Группа радиокнопок (<input type='radio'>) определяется путем присвоения каждой из радиокнопок
в группе атрибута name с одинаковым значением. Радиокнопкам также необходим атрибут value,
значение которого представляет собой данные, отправляемые на сервер.

  Элемент <select> должен иметь атрибут name, чтобы данные имели идентификатор для отправки в
базу данных с выбранным значением.
Если атрибут value отсутствует в элементе <option>, то значение будет взято из текстового
содержимого элемента option.

  Элемент <datalist> работает в сочетании с элементом <input>, чтобы предоставить пользователю
возможные варианты, при этом давая ему возможность вводить собственное значение в поле ввода.
Элемент datalist должен иметь id. И это значение id должно быть равно аттрибуту list у <input>.
Атрибут name помещается в элемент <input>:
        
          
<label for='choice'>What's your favourite ice cream flavor?</label>
<input list='ice-cream-flavors' id='choice' name='ice-cream-choice' type='text'/>
<datalist id='ice-cream-flavors'>
  <option value='Chocolate'></option>
  <option value='Coconut'></option>
  <option value='Mint'></option>
</datalist>
          
        

  Атрибуты type='submit' и type='reset' можно применять к элементу <input> или <button> 
в форме. Разница между этими двумя элементами заключается в том, что, поскольку элемент 
<button type='submit'> имеет закрывающий тег, то можно помещать внутрь него другие элементы.

  Aтрибут required можно добавить к любому элементу ввода формы: <input>, <textarea> и <select>.
Для группы радиокнопок (некоторые <input type='radio'> с одинаковыми значениямии name) мы можем
задать required только для какого то одного <input type='radio'>.
  С помощью HTML можно сделать один <input type='checkbox'> обязательным, добавив required.
Однако, если вы хотите, чтобы ваши пользователи выбирали хотя бы один checkbox из группы
checkbox'ов, то нужен JS.

a11y


  ARIA stands for Accessible Rich Internet Applications and is a set of attributes
that define ways to make web content and web applications more accessible to people
with disabilities.
  Например если в ссылку на соц. сеть вложена картинка, то она не будет видна слепым
людям, полагающимся только на скринридер ⇒ нужно использовать aria-оттрибуты.


  Links that open in a separate tab without informing a screenreader user can be
confusing too.
  The value for an aria-label is a string of text that labels the purpose and behaviour
of the element. For example:
        
          
<a
  href='https://twitter.com/CodersInstitute'
  target='_blank'
  aria-label='Go to our twitter page (opens in new tab)' 
>
    Twitter
</a>
          
        


  For images, we have already talked about using the alt attribute on an <img> element,
to describe the image. Without an alt attribute, a screenreader will read out the name
of the image file that is being loaded, which is rarely descriptive enough.

  Suppose you use a background-image with css, and the image is an integral part of the
information you are providing to the user. In that case, an aria-label attribute should
be added to the element to describe the image. The reason for this is a background image
is not applied using an img element, so there would be no alt attribute available to it
in this case.


  The aria-labelledby attribute identifies the element (or elements) that labels the
element it is applied to.
        
The following example uses aria-labelledby to provide an accessible
name for a checkbox input by using the text content of a sibling element:
          
<span
  role='checkbox'
  aria-checked='false'
  tabindex='0'
  aria-labelledby='tac'></span>
<span id='tac'>I agree to the Terms and Conditions</span>
